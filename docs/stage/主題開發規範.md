# Markdown 主題開發規範 (Guide)

為了確保 Markdown Live Previewer 的主題既具備品牌辨識度，又能提供卓越的閱讀體驗，請在開發新主題時遵循以下規範。

---

## 1. 易讀性優先 (Accessibility First)

### 🎨 對比度要求
- **主內文**：對比度必須符合 WCAG 2.1 AA 標準 (至少 4.5:1)。
- **輔助文字**：(如標題前綴、行號) 至少需達到 3:1。
- **背景色**：避免使用純飽和色作為背景（如純藍或純綠），建議使用帶有微量主色調的「灰階色」(Slate/Gray)。
- **深/淺色模式**：在深色/淺色模式下，所有文字與圖表元素都應保持高對比度，確保在明暗對比強烈的環境下依然清晰易讀。
- **列印**：在列印模式下，要忠實呈現預覽的內容，而非僅僅是一份修改後的頁面。

### 🖋️ 字體選擇
- **學術/正式**：優先選用襯線體 (Serif)，行高建議 1.6 - 1.8。
- **現代/筆記**：選用無襯線體 (Sans-serif)，增加字距 (Letter-spacing: -0.01em)。
- **技術/代碼**：強制使用等寬字體 (Monospace)，且需考慮中文與英文混排的對齊感。

---

## 2. 品牌美學 (Aesthetics)

- **微動畫**：主題切換時，請確保過渡效果平滑（建議 0.3s - 0.5s）。
- **視覺錨點**：每個主題應有一個明確的「特點」，例如：
    - `Academic`: 標題居中。
    - `Developer`: `##` 前綴的可視化。
    - `Minimal`: 大量的留白。

---

## 3. 列印適配 (Print Adaptation)

當使用者將文檔匯出為 PDF 時，主題不應被完全抹除。

### 🔄 列印轉化邏輯
1.  **背景處理**：除非主題特色要求，否則列印時建議強制背景為白色，文字為黑色。
2.  **暗色主題轉化**：若為深色主題，列印時應反轉顏色（Invert），而非僅僅隱藏背景。
3.  **邊框與間距**：列印模式下應移除所有 Box-shadow，並將 Border 轉化為極細的灰色實線 (0.5pt)。

### 🛠️ 實作方法
在主題 CSS 中使用 `@media print` 區塊來覆寫全局設定：
```css
@media print {
    .theme-your-name.prose {
        background-color: white !important;
        color: black !important;
    }
}
```
